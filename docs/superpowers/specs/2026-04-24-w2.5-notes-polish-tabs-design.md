# W2.5 — Notes polish + tab system

**Status:** design locked 2026-04-24 · ready for implementation plan
**Phase:** Backend Wiring follow-up. Consumes W2's Notes wiring as baseline; no Rust changes except a possible `get_node_ancestors` convenience command.
**Predecessor:** [2026-04-23-w2-notes-wiring-design.md](2026-04-23-w2-notes-wiring-design.md) (shipped 2026-04-24).

---

## 1. Goal

Close the "looks bare" and "font too small" feedback on the W2 Notes
surface and land the tab system needed to make the editor actually
usable for multi-doc work. Adds the Obsidian-like chrome
(breadcrumb + editable title + collapsible properties panel) around
the existing CM6 editor, plus in-editor placeholder for the slash /
wikilink hints. Fixes the `@dnd-kit` drag-offset visual bug flagged
during use.

After W2.5, the Notes surface should read as feature-complete for a
user who opens it expecting "Obsidian but local-first with AI
incoming." Remaining gaps (`/voice` / `/database` / `/embed` slash
commands, conflict-banner diff view, cross-restart tab persistence)
stay deferred to later phases.

## 2. Scope

### Included
1. **Typography bump** across tree, tree controls, backlinks.
2. **Drag offset fix** on the tree DnD source row.
3. **Tab system** — per-tab state (nodeId, preview/permanent, dirty,
   scrollTop), tab strip UI, keyboard shortcuts.
4. **Editor title bar** — emoji icon + editable title input.
5. **Breadcrumb strip** — clickable ancestor chain with
   middle-collapse on overflow.
6. **CM6 placeholder** replacing the earlier "slash hint strip"
   idea — uses `@codemirror/view`'s `placeholder` extension.
7. **Properties panel** — collapsed-by-default; editable icon + tags;
   everything else read-only.
8. **Overflow-safe rendering** on every text surface that can
   contain user strings (title, tab label, tree row, breadcrumb,
   backlinks item, tag chip).

- Tab persistence across app restart (in-memory only for W2.5).
- Drag-to-reorder tabs in the tab strip (close-and-reopen works).
- Multi-pane / split view — CLAUDE.md Deferred list.
- User-editable frontmatter fields other than icon + tags.
- `/voice`, `/database`, `/embed` slash commands (still W2.5-or-later
  per the PLAN.md backlog).
- Conflict-banner diff view.
- Persistent per-tab scroll position across app restart.

## 4. Locked decisions (from brainstorming)

1. **Tab system scope**: full — tab strip + per-tab state + keyboard
   shortcuts. Not single-tab-only-shell.
2. **Properties panel scope**: icon + tags only. Everything else
   read-only. Tags stored as `properties_json.tags: string[]`.
3. **Preview vs permanent tab semantics** (VS Code pattern):
   - Tree plain-click → opens in a PREVIEW tab (italic label).
     Next tree plain-click REPLACES the preview tab's nodeId.
   - Tree right-click → "Open in new tab" → always opens a
     PERMANENT tab (regular weight label). Doesn't replace.
   - Editing a preview tab (any user `docChanged`) PROMOTES it to
     permanent.
   - Wikilink plain-click inside editor → same as tree plain-click
     (preview-replace).
   - Wikilink Cmd/Ctrl-click → opens in new permanent tab.
4. **Tab invariants**: every tab has a `nodeId` (no empty tabs).
   "+ New tab" button in the strip creates a fresh root doc (same
   path as Cmd+N) so the new tab always has a node.
5. **Tab persistence**: in-memory only. Closing the app loses the tab
   layout; reopening starts with a single empty-state card until the
   user opens a note.
6. **Slash/wikilink hint**: CM6 `placeholder` inside the editor, not
   a separate strip. Shows only when doc is empty and editor
   unfocused.
7. **Breadcrumb overflow**: middle-collapse. Keep root + leaf always
   visible; collapse middles with `…`.
8. **Title overflow**: ellipsis when unfocused, horizontal scroll
   when focused (native `<input>` behavior with
   `text-overflow: ellipsis`).
9. **Tab label overflow**: `max-width: 180px` + ellipsis + native
   `title` tooltip.
10. **Typography targets**: tree + backlinks rows → 14px;
    tree buttons → 12px; tabs → 13px medium; breadcrumb → 11px
    muted; title input → 28px light.
11. **Drag offset fix**: `transform: isDragging ? undefined :
    CSS.Transform.toString(transform)` on `SortableRow`.

### 5.2 State ownership

| State | Owner | Shape |
|---|---|---|
| Tab strip (list, active, preview/permanent, dirty) | `NotesView` via `useReducer(tabsReducer)` | §7 |
| CM6 doc body | `MarkdownEditor` via `EditorState.doc` (Approach B — unchanged from W2) | markdown string |
| Conflict state | `MarkdownEditor` via `useReducer(conflictReducer)` (unchanged) | §6.2 of W2 spec |
| Active node id (derived) | `NotesView.state.tabs.find(t => t.id === activeTabId)?.nodeId` | string / null |
| Node meta (name, icon, properties) | `MarkdownEditor` per-tab via `nodeMetaRef` (unchanged) | internal ref |
| Tab dirty flag | Propagated UP from MarkdownEditor via `onDirtyChange?: (dirty: boolean) => void` prop | boolean |
| Tab scroll position | Propagated UP from MarkdownEditor via `onScrollChange?: (top: number) => void` on scroll | number (no persistence) |
| Properties panel collapsed | `PropertiesPanel` local state (collapsed by default) | boolean |

### 5.3 Key data flows

**Tree click → preview-replace OR new permanent tab:**
```
user clicks tree row
  → NotesView's Tree.onSelect callback
    → dispatch({ type: 'OPEN_PREVIEW', nodeId })
    → tabsReducer either replaces the active preview tab's nodeId
      or creates a new preview tab (if active tab is permanent or dirty)
      or creates a fresh preview tab (if no active tab)
```

**Tree right-click → "Open in new tab":**
```
user right-clicks tree row → HerOSMenu opens
  → user chooses "Open in new tab"
    → dispatch({ type: 'OPEN_IN_NEW_TAB', nodeId })
    → tabsReducer appends a new permanent tab and activates it
```

**Editor user edit → preview → permanent promotion:**
```
user types in the CM6 editor
  → MarkdownEditor.autosavePlugin.update listener fires
    → onDirtyChange(true) → NotesView dispatches MARK_DIRTY + PROMOTE_PREVIEW
    → tab label flips from italic to regular weight
```

**Save succeeds → clean dirty flag:**
```
autosave fires → commands.updateNode ok → conflict-reducer SAVE_OK
  → MarkdownEditor calls onDirtyChange(false)
    → NotesView dispatches MARK_CLEAN(tabId)
```

**Wikilink click inside editor:**
```
user clicks a [Name](node://uuid) link
  → nodeLinkClickPlugin event handler
    → if event.metaKey || event.ctrlKey:
         onNodeLinkClick(nodeId, { newTab: true })
         → dispatch({ type: 'OPEN_IN_NEW_TAB', nodeId })
       else:
         onNodeLinkClick(nodeId, { newTab: false })
         → dispatch({ type: 'OPEN_PREVIEW', nodeId })
```

### 5.4 MarkdownEditor remount key + scroll restoration

NotesView renders `<MarkdownEditor>` with a **compound key**:

```tsx
<MarkdownEditor
  key={`${activeTab.id}:${activeTab.nodeId}`}
  nodeId={activeTab.nodeId}
  initialScrollTop={activeTab.scrollTop}
  onScrollChange={(top) => dispatch({ type: 'SET_SCROLL', tabId: activeTab.id, scrollTop: top })}
  onDirtyChange={(dirty) => {
    dispatch({ type: 'MARK_DIRTY', tabId: activeTab.id, dirty })
    if (dirty && activeTab.preview) dispatch({ type: 'PROMOTE_PREVIEW', tabId: activeTab.id })
  }}
  onNodeLinkClick={(nodeId, opts) => dispatch(
    opts.meta ? { type: 'OPEN_IN_NEW_TAB', nodeId } : { type: 'OPEN_PREVIEW', nodeId }
  )}
/>
```

Compound key behavior:
- Tab SWITCH (user clicks different tab) → `activeTab.id` changes → key changes → remount with new nodeId.
- Preview-REPLACE (tree click on same preview tab) → `activeTab.nodeId` changes → key changes → remount with new nodeId (same tab id).
- User edits within the same tab → key unchanged → no remount.

Scroll restoration (inside `MarkdownEditor`'s view-build effect, after
`new EditorView({...})`):

```ts
if (initialScrollTop && initialScrollTop > 0) {
  requestAnimationFrame(() => {
    if (parentRef.current) parentRef.current.scrollTop = initialScrollTop
  })
}
```

`onScrollChange` throttled via `requestAnimationFrame` to once per
frame (`parentRef.current.addEventListener('scroll', …)`).

## 7. Tab state machine in detail

### 7.1 Shape + actions

```ts
export interface Tab {
  id: string                    // unique tab id (e.g. crypto.randomUUID())
  nodeId: string                // which node this tab shows
  preview: boolean              // true = italic label, auto-replaced; false = permanent
  dirty: boolean                // has unsaved user edits; set via onDirtyChange
  scrollTop: number             // last-known scroll position (in-memory only)
}

export interface TabsState {
  tabs: Tab[]
  activeTabId: string | null
}

export const initialTabsState: TabsState = {
  tabs: [],
  activeTabId: null,
}

export type TabsAction =
  | { type: 'OPEN_PREVIEW'; nodeId: string }
  | { type: 'OPEN_IN_NEW_TAB'; nodeId: string }
  | { type: 'SWITCH_TAB'; tabId: string }
  | { type: 'CLOSE_TAB'; tabId: string }
  | { type: 'CLOSE_ACTIVE' }
  | { type: 'PROMOTE_PREVIEW'; tabId: string }
  | { type: 'MARK_DIRTY'; tabId: string; dirty: boolean }
  | { type: 'SET_SCROLL'; tabId: string; scrollTop: number }
  | { type: 'SWITCH_TO_INDEX'; index: number }  // Cmd+1..9
```

### 7.2 Key transitions

**OPEN_PREVIEW(nodeId):**
```
if (!activeTab) {
  // no tabs yet → open a new preview tab
  newTab = { id: uuid(), nodeId, preview: true, dirty: false, scrollTop: 0 }
  tabs.push(newTab)
  activeTabId = newTab.id
} else if (activeTab.preview && !activeTab.dirty) {
  // replace active preview tab's nodeId in place (keeps tab id stable)
  activeTab.nodeId = nodeId
  activeTab.scrollTop = 0
} else {
  // active is permanent or dirty → open a new preview tab
  newTab = { id: uuid(), nodeId, preview: true, dirty: false, scrollTop: 0 }
  tabs.push(newTab)
  activeTabId = newTab.id
}
```

**OPEN_IN_NEW_TAB(nodeId):**
```
newTab = { id: uuid(), nodeId, preview: false, dirty: false, scrollTop: 0 }
tabs.push(newTab)
activeTabId = newTab.id
```

**PROMOTE_PREVIEW(tabId):**
```
tab = tabs.find(t => t.id === tabId)
if (tab && tab.preview) tab.preview = false
// Called when user edits the preview tab's body OR double-clicks its tab label.
```

**CLOSE_TAB(tabId):**
```
i = tabs.findIndex(t => t.id === tabId)
if (i === -1) return state
tabs.splice(i, 1)
if (activeTabId === tabId) {
  // activate neighbour: prefer the one to the right, fall back to left
  const next = tabs[i] ?? tabs[i - 1] ?? null
  activeTabId = next?.id ?? null
}
```

**Dirty-close guard:** the reducer allows closing a dirty tab.
The UI layer (NotesView) is responsible for:
1. Flushing the MarkdownEditor's pending save via `saver.flush()`
2. THEN dispatching `CLOSE_TAB`.

If the flush fails (conflict), the UI shows the conflict banner
rather than closing the tab; the user must resolve before closing.

**MARK_DIRTY(tabId, dirty):**
```
tab = tabs.find(t => t.id === tabId)
if (tab && tab.dirty !== dirty) tab.dirty = dirty
// Called from MarkdownEditor.onDirtyChange.
// If user edits a preview tab: dispatch PROMOTE_PREVIEW first, then MARK_DIRTY.
```

### 7.3 Invariants

- **Exactly one active tab when tabs.length > 0.** Enforced by reducer.
- **Every tab's nodeId is a valid WorkspaceNode id.** Enforced at dispatch time (NotesView only dispatches with ids it got from Tree / wikilink / daily-note commands).
- **tabs.length === 0 ⇒ activeTabId === null.** Enforced by reducer.
- **At most one preview tab at a time** is NOT enforced — the reducer design produces this naturally (OPEN_PREVIEW either replaces the active preview or appends a new one when active is permanent, never appends alongside another preview). A test pins this.
- **Every tab has a unique id.** `crypto.randomUUID()` guarantees.

### 7.4 Test coverage (~15 tests in `tabsReducer.test.ts`)

1. Empty initial state.
2. OPEN_PREVIEW on empty → new preview tab, activated.
3. OPEN_PREVIEW with active preview + clean → replaces nodeId in place, tab id stable.
4. OPEN_PREVIEW with active preview + dirty → new preview tab, old tab stays.
5. OPEN_PREVIEW with active permanent → new preview tab.
6. OPEN_IN_NEW_TAB → always appends a permanent tab, activates it.
7. PROMOTE_PREVIEW flips flag; no-op if already permanent.
8. MARK_DIRTY flips flag; no-op if unchanged.
9. CLOSE_TAB in middle activates next-right neighbour.
10. CLOSE_TAB at end activates previous (left).
11. CLOSE_TAB last tab → activeTabId === null.
12. CLOSE_TAB on non-active doesn't change activeTabId.
13. SWITCH_TAB to existing id → activeTabId updates.
14. SWITCH_TAB to missing id → no-op.
15. SWITCH_TO_INDEX clamps to tabs.length - 1.

## 8. Keyboard contracts

Added in AppShell. All gated on `!e.target.closest('input, textarea, .cm-editor')` where appropriate so typing in a field doesn't trigger them (except Cmd+W, which always closes the active tab regardless of focus).

| Shortcut | Action |
|---|---|
| `Cmd/Ctrl+N` | Create new root doc → opens in new preview tab (existing, W2) |
| `Cmd/Ctrl+T` | Create new blank doc + new permanent tab (differs from Cmd+N — explicitly "open a new tab") |
| `Cmd/Ctrl+W` | Close active tab (flush save first; if conflict, banner shows, tab stays) |
| `Cmd/Ctrl+1..9` | Switch to tab at that 1-based index (clamps to last tab) |
| `Cmd/Ctrl+Shift+J` | Today's daily note → opens in new preview tab (existing) |

Collisions audited against CLAUDE.md Keyboard Contracts:
- `Cmd+T` is currently unbound — safe.
- `Cmd+W` on macOS is native "close window". We're overloading it to close tab, matching Obsidian / VS Code / browser behavior. On last tab, Cmd+W does NOT close the window — the empty-state card shows. User closes window via macOS red dot or `Cmd+Q`. Document in CLAUDE.md Keyboard Contracts update.
- `Cmd+1..9` is currently unbound — safe.

### 9.6 `ancestors.ts`

```ts
export async function getAncestors(
  leafId: string,
  commands: typeof import('../bindings').commands,
): Promise<WorkspaceNode[]> {
  const chain: WorkspaceNode[] = []
  let currentId: string | null = leafId
  while (currentId) {
    const res = await commands.getNode(currentId)
    if (res.status !== 'ok' || !res.data) break
    chain.unshift(res.data)                      // prepend — build root-to-leaf
    currentId = res.data.parent_id
    if (chain.length > 64) break                 // cycle guard / depth cap
  }
  return chain
}
```

Plus a small session-memoized hook:

```ts
export function useAncestors(nodeId: string | null): {
  ancestors: WorkspaceNode[]
  loading: boolean
  error: string | null
}
```

Internally caches by `nodeId` in a module-scoped `Map<string, WorkspaceNode[]>`.
Cache is cleared when any `updateNode` succeeds for a node in a cached
chain (or when `refreshToken` in NotesView bumps — simpler; flush the
whole cache on any tree mutation).

**Alternative**: introduce a Rust command `get_node_ancestors(id): Vec<WorkspaceNode>` that does the walk server-side in one query. Cleaner but adds bindings regen and tauri-specta round-trip. Decision: **start client-side** — acceptable for depths < 10 (most vaults). Revisit if the N+1 fetch shows up in profiling.

## 14. Risks

1. **Tab dirty-then-close UX**. If user hits Cmd+W on a dirty tab, we
   flush-save first. If flush returns VAULT_CONFLICT, the conflict
   banner shows and the tab stays open. But the user pressed Cmd+W
   with intent to close — if they ignore the banner and hit Cmd+W
   again, should we force-close and lose their edits? Decision:
   **never force-close**. Cmd+W on a conflicted tab re-shows the
   banner; user must choose Reload or Keep-mine, which resolves the
   save, at which point Cmd+W works. Document this in the tab strip's
   close-hover tooltip when dirty: `Unsaved — resolve conflict first`.

2. **Session-memoized `ancestors` cache invalidation**. The simple
   "flush all on any tree mutation" strategy is correct but wasteful.
   A more targeted invalidation (only flush ancestors of the mutated
   node) requires a reverse index — not worth building in W2.5.
   Acceptable cost because ancestor walks are bounded at depth 64.

3. **Emoji picker UX for title bar + properties panel**. Curated list
   of 20 emojis is a stopgap. Obsidian uses a proper emoji-mart
   picker. Calling this out as known-not-great; will get replaced
   when an emoji-mart-style picker lands (W6 or later).

4. **Double-click tab to promote**. Some users will try to double-
   click the tab label to rename it (Obsidian pattern). We reserve
   double-click for promote-preview. Renaming happens only via the
   title bar input. Live with the collision; add a tooltip if users
   report confusion.
