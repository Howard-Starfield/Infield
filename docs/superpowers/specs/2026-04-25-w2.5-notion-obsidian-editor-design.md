# W2.5 — Notion/Obsidian-style Markdown Editor

**Date:** 2026-04-25
**Phase:** W2.5 (slot between W2 Notes wiring and W3 Search)
**Status:** Design approved, ready for implementation plan

## Context & Problem

W2 wired the Notes surface to the Rust backend: `getNode` / `updateNode` round-trip, vault-write conflict guard, autosave debounce, `[[` wikilink autocomplete, `/` slash commands. Functionally complete.

Visually, the editor is still a plain CodeMirror 6 source view:

- Markdown body renders as raw source — `# Heading` is the literal hash and a normal-size font, `- [ ]` is literal text not a clickable checkbox, `**bold**` shows the asterisks.
- The `/` slash menu is a flat CM6 autocomplete list — no icons, no category grouping, no visible shortcut hints (the `shortcutHint` field exists in [src/editor/slashCommands.ts](../../src/editor/slashCommands.ts) but isn't drawn).

The "Notion/Obsidian feel" users expect comes from two distinct features layered on top of the same markdown source:

1. **Live Preview block rendering** — markers (`*`, `**`, `~~`, `` ` ``, `# `, `> `) hide when the caret is on a different line, restore when the caret enters. Headings scale visually. Tasks become real checkboxes. Dividers render as `<hr>`.
2. **Polished slash menu** — icon column, category section headers, right-aligned shortcut chip, two-line label/description rows.

Backend is unchanged. This is a frontend-only phase touching the CodeMirror 6 extension stack inside [src/components/MarkdownEditor.tsx](../../src/components/MarkdownEditor.tsx).

## Non-Goals (Explicitly Deferred)

- **Pipe table rendering and cell editor.** Tables, kanban boards, and calendar views all share the same underlying concept (a Notion "database block"). Handy already reserves `node_type='database'` for this (Rule 11, [docs/architecture/vault-database-storage.md](../../architecture/vault-database-storage.md)). They will be designed together as a later phase covering the W4 Database surface plus its alternate views.
- **Drag handles per block.** Notion-only feature, not in Obsidian. Defer.
- **Callouts (`> [!note]`), math (`$x$`), Mermaid, footnotes, block references.** Tier-3 features each worth their own mini-phase.
- **Syntax highlighting inside fenced code blocks.** Fenced code renders as a styled block in W2.5; per-language highlighting via `@codemirror/language-data` is a polish item for later.
- **Column resize, sort/filter, column types** — already in CLAUDE.md "Deferred — Do Not Implement in v1" list.

## Scope — Two Sub-phases

### W2.5a: Slash menu polish

**Visible changes** when the user types `/` at line start:

- **Icon column** — 18×18 SVG to the left of each command label. Source: `lucide-react` (matches Obsidian, which uses Lucide natively; tree-shakes to ~12 icons used).
- **Section headers** — non-selectable rows grouping commands by `category`:
  - **Basic blocks** — `h1` `h2` `h3` `ul` `ol` `todo` `quote` `divider`
  - **Code** — `code`
  - **Handy** — `link` `today`
- **Shortcut chip** — right-aligned, muted, renders `cmd.shortcutHint` (e.g. `⌘1`) when present. `<kbd>`-style with `var(--surface-container)` background.
- **Two-line layout** — bold label on top, muted description below.
- **Selected row** — left accent bar in `var(--heros-brand)` (Notion-style), keeping the existing `[aria-selected]` background.
- **Removed:** `Table` slash command (deferred to the table/kanban/calendar phase). One-line revert when that phase lands.

**Implementation:**

- CodeMirror's `autocompletion()` config accepts `addToOptions: { render, position }[]`. Three renderers: icon cell, label-stack cell, shortcut cell. Each gets the `Completion` and returns a DOM `Node`.
- For section headers, synthesize "header" `Completion` entries with `apply: () => {}` and a custom `type: 'section-header'`. The renderer detects this `type` and emits an `<li role="presentation">` with no click handler.
- Grouping happens in `slashCompletion.ts` before the `Completion[]` is returned: filter by query → bucket by category → flatten with header rows interleaved.

**Files:**

- `src/editor/slashCompletion.ts` — grouping + section-header synthesis (~30 LOC added)
- `src/editor/slashCommands.ts` — remove `Table` from `tier1SlashCommands`
- `src/components/MarkdownEditor.tsx` — pass `addToOptions` to `autocompletion()`
- `src/styles/notes.css` — `.cm-md-slash-row`, `.cm-md-slash-icon`, `.cm-md-slash-shortcut`, `.cm-md-slash-section`, `.cm-md-slash-accent`
- `package.json` — add `lucide-react`

**Tests:**

- `src/editor/__tests__/slashCompletion.test.ts` — extend existing fixtures with grouping assertions
- New: snapshot of the rendered popup DOM via vitest + jsdom (one happy-path render)

**Effort:** half a day.

### W2.5b: Live Preview block rendering

**Architecture:**

A single CodeMirror 6 view plugin (`livePreviewPlugin`) owns a `DecorationSet`, rebuilt on:

- `update.docChanged`
- `update.selectionSet`
- `update.viewportChanged`

It walks the `@lezer/markdown` GFM tree (already produced by the existing `markdown({ base: markdownLanguage, extensions: [GFM] })` extension) and emits decorations of three kinds:

- `Decoration.replace` — swap range for a widget. Used for: task checkboxes, horizontal rule.
- `Decoration.mark` — add CSS class to range. Used for: heading scale, soft-marker hiding, inline bold/italic/strike/code styling, link styling.
- `Decoration.line` — add class to whole line. Used for: heading line, blockquote line.

**The single rule producing the Obsidian feel — caret-line marker visibility:**

At decoration-build time, compute `caretLine = state.doc.lineAt(state.selection.main.head).number`. For each markdown construct, if its source range overlaps the caret line, the "marker hidden" decoration is skipped (markers stay visible, dimmed via `cm-md-marker`). Otherwise the marker range gets `cm-md-hidden` (`display: none`).

When the user has a non-empty selection, every line spanned by the selection counts as a caret line — the user is selecting and wants source.

**Per-block rendering contract:**

| Lezer node | Off-caret-line render | On-caret-line render | Source preserved |
|---|---|---|---|
| `ATXHeading1`–`ATXHeading6` | Scaled font (h1: 1.9em, h2: 1.55em, h3: 1.3em, h4–6: 1.15em) via `cm-md-h{n}` line class. `# ` markers hidden. | Same scale + markers visible/dimmed. | `# Heading` |
| `Emphasis` | `*text*` → italic; `*` markers hidden | Italic + markers visible/dimmed | `*text*` |
| `StrongEmphasis` | `**text**` → bold; markers hidden | Bold + markers visible/dimmed | `**text**` |
| `Strikethrough` (GFM) | `~~text~~` → strike; markers hidden | Strike + markers visible/dimmed | `~~text~~` |
| `InlineCode` | `` `code` `` → chip-styled; backticks hidden | Chip + backticks visible | `` `code` `` |
| `FencedCode` | Padded block w/ `cm-md-code-block` line class. Fence lines (` ``` `) **always visible** to keep editing them sane. No syntax highlighting in W2.5. | Same | unchanged |
| `Blockquote` | `> ` marker hidden, line gets left-border bar via `cm-md-blockquote` line class | Marker visible/dimmed + bar | `> text` |
| `BulletList` `ListItem` | `- ` / `*` replaced via `Decoration.replace` with a styled bullet `•`; nesting indent preserved | Bullet visible/dimmed | `- text` |
| `OrderedList` `ListItem` | `1.` `2.` kept (numbers are meaningful) but styled muted | Same | `1. text` |
| `Task` (GFM `- [ ]` / `- [x]`) | Replace `[ ]` / `[x]` with `TaskCheckboxWidget` — real `<input type="checkbox">`. Click toggles `[ ]` ↔ `[x]` via atomic `view.dispatch`. Bullet `-` becomes a styled bullet too. | Widget **always** rendered (even on caret line) so the checkbox stays clickable. The `- ` bullet marker still toggles per the bullet rule above. | `- [ ] text` / `- [x] text` |
| `HorizontalRule` (`---`) | `Decoration.replace` → `<hr>` widget styled with HerOS tokens | `---` source visible | `---` |
| `Link` (markdown + GFM autolink) | `[text](url)` shows just `text` styled as link; `()` and brackets hidden | All visible | `[text](url)` |
| `node://` link | Untouched — `nodeLinkClickPlugin` (already shipped) handles these | Untouched | unchanged |

**Edge cases baked into the design:**

1. **Multi-line constructs.** Fenced-code fences and blockquote `>` markers stay visible always; only single-line markers (`*`, `**`, `~~`, `` ` ``, `[`/`](...)`) hide off-line. Hiding multi-line markers makes editing them jarring.
2. **Selection.** Non-empty selection treats every line in the selection as caret-line.
3. **IME composition.** Skip decoration rebuild while `update.transactions.some(tr => tr.isUserEvent('input.type'))` is firing for an active composition. Prevents widget flicker for Chinese/Japanese/Korean input.
4. **Inside fenced code.** Lines inside a `FencedCode` block get no markdown decorations — the fence content isn't markdown.
5. **Task widget always rendered.** Unlike inline markers (`*`, `**`, etc.), the `[ ]`/`[x]` checkbox widget is *always* present, even on the caret line. Hiding it would mean the user couldn't click to toggle while editing the same line. This matches Obsidian's behaviour. Source `[ ]` is never visible — it's permanently replaced by the widget. The rest of the task line (bullet marker, text content) follows the normal caret-line rule.

## Performance Budget

| Metric | Target |
|---|---|
| Decoration rebuild on caret move (typical 500-line doc) | <1ms |
| Decoration rebuild on caret move (worst-case 5,000-line doc) | <2ms |
| Initial decoration build on doc open (5,000-line doc) | <10ms |
| Frame budget while scrolling at 60fps | held |
| Slash menu open → first paint | <16ms (one frame) |

Lezer tree walk is incremental via `RangeSetBuilder`; only viewport-visible blocks materialize widgets; off-screen content walks the tree but emits no DOM. Caret-move recompute is O(viewport-nodes) — proven in similar Lezer setups.

## Risks & Mitigations

- **Risk: decoration rebuild storm on every keystroke.** Mitigation: `update.docChanged || update.selectionSet || update.viewportChanged` is the only trigger; nothing else rebuilds. Lezer tree is already incremental from CM6.
- **Risk: widget interactivity (checkbox click) breaks undo/redo.** Mitigation: dispatch the toggle as a single user-event transaction with `userEvent: 'input.toggle'`. Undo replays it as one step. Tested in `livePreview.test.ts`.
- **Risk: caret-line model fights wikilink autocomplete or voice memo pill.** Mitigation: those plugins already use their own decoration sets and don't touch the markdown-marker ranges. No overlap. Verify in the manual checklist.
- **Risk: `addToOptions` is a lower-traffic CM6 API and might fight us.** Fallback: a custom render override on the `autocompletion` config — same end result, ~40 extra LOC. Document the fallback in the implementation plan.
- **Risk: lucide-react adds noticeable bundle weight.** Mitigation: tree-shaken imports (`import { Heading1 } from 'lucide-react'`) ship only the ~12 icons used; total addition <5kb gzipped.

## Future Phases This Unblocks

- **Tables / kanban / calendar phase** — when this lands, the Live Preview decoration model is reusable for any new block type. Adding a `TableWidget` becomes "one new widget class + one Lezer-node case in `livePreview.ts`."
- **Callouts (`> [!note]`)** — extends the `Blockquote` rendering case; small.
- **Math (`$x$`) and Mermaid** — both are widget-replace cases on existing Lezer nodes once a renderer (KaTeX, Mermaid) is wired.
- **Block references (`[[note#^block-id]]`)** — extends wikilink completion + node-link click plugin; orthogonal to this design.
