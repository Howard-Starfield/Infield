Nested Space Tree — Two Layout Approaches

Spaces are folders. Notes and database views are files. Both can nest freely. Which tree model feels right?

A — Unified Tree (AppFlowy-style) ✦ Recommended
▶ 📁 Work space
▶ 📁 Projects space
⊞ Task Tracker grid
📋 Sprint Board board
📄 Spec Doc
📄 Meeting Notes
▶ 📁 Personal space
Unsorted
📄 Quick Note
+ New Space

Spaces (📁) and notes/views (📄⊞📋) live in the same tree. Drag anything anywhere. "Unsorted" holds orphan notes.

B — Two-Panel (spaces rail + notes tree)
SPACES
Work
Personal
Archive
+ Add
WORK
▶ 📁 Projects
⊞ Task Tracker
📋 Sprint Board
📄 Meeting Notes

Spaces in a fixed left rail, notes tree on the right. Simpler but drag-to-space is clunky (must drag across panels).

How DnD works in A: Drag any note or space onto another space to move it in. Drag to the "Unsorted" section to orphan it. Drag a space onto another space to nest it. All powered by @dnd-kit (already installed).
