Unified Workspace Model — One Table, Three Node Types

workspace_nodes is the single source of truth. Views are projections over node data.

workspace_nodes — unified table
id parent_id node_type name icon properties (JSON) body (rich text) position
n1 null document Meeting Notes 📄 — rich text body 0
n2 null database Project Tracker 📊 {fields: [...]} — 1
n3 n2 row Q2 Campaign planning 📋 {status: "In Progress", priority: "High", ...} rich text body 0
n4 n2 row Website redesign 📋 {status: "To Do", priority: "High", ...} — 1
n5 n2 row Mobile app v2 📋 {status: "Done", priority: "Medium", ...} — 2
node_views — views (lightweight, no row data)
id node_id (FK) name layout position view_options (JSON)
v1 n2 Board board 0 {groupByField: "status"}
v2 n2 Grid grid 1 {}
Key insight: Views (node_views) are attached to database nodes. They are projections — they query the workspace_nodes table to find child row nodes and display them. The view never holds row data; it just knows which layout to use and how to group/filter/sort.
