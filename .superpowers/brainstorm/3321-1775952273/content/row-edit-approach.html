Row Detail Panel — how should editing work?

When a user clicks a row to view or edit it

A

Slide-in panel from right

Row detail slides in from the right edge (280ms ease, no bounce). The database view stays visible behind it. Click outside or press Escape to close. Like Notion.

B

Inline modal / dialog

A centered modal dialog with a backdrop overlay. Blocks interaction with the view behind until closed. More emphatic interruption.

C

Inline expansion (expand in place)

The row expands downward within the view, pushing other rows down. No sliding, no overlay. Like a table row expanding. Keeps context visible.

D

Full-page takeover

The entire database view is replaced by the row editor. Most space for fields, but loses view context entirely.
